📖 User manual

What is this?

Second Brain is a personal workspace that lives entirely in your browser. You organize projects, and inside each project a tree of folders and files. Every file holds a piece of HTML that gets rendered for you.

Projects (this screen)

  • + New project creates a project. Click a card to open it.
  • The ⋯ on a project card lets you open, rename, export or delete it.

The tree (left sidebar)

  • Click a file → its HTML is rendered full-size on the right.
  • Click a folder → the right side renders everything inside it: each file becomes a block, and each subfolder becomes its own block containing its files (and its subfolders, recursively).
  • The ▸ / ▾ caret only expands or collapses a folder in the tree — it doesn't change the selection.
  • The ⋯ menu on any item: new file / new subfolder (folders), edit HTML and status (files), rename, move, reorder, delete.
  • The dot next to a file is its status — click it to cycle: to do → in progress → done.

Creating files & folders — where do they go?

  • The + Folder and + File buttons at the bottom of the sidebar create inside the currently selected folder.
  • If nothing is selected (or a file is selected), they are created at the project root.
  • To create inside a specific folder regardless of selection, use that folder's ⋯ menu → + New file / + New subfolder.
  • Placed something in the wrong spot? Use ⋯ → Move… to relocate it anywhere in the tree.

Editing file content

  • Open a file and press ✏️ Edit HTML (or ⋯ → Edit HTML). Paste or write any HTML; 👁 Preview shows the result before saving.
  • Files render in a sandboxed frame, so scripts inside your HTML run safely per file. Drag the bottom edge of a rendered block to resize it.

Themes

🎨 Theme (inside a project) sets that project's colors — pick a preset or tune each color individually.

Saving, backup & AI

  • Everything saves automatically to this browser's local storage. It never leaves your machine — but clearing browser data (or switching browser/device) loses it, so export regularly.
  • 💾 Export all downloads the whole workspace as JSON; 💾 Export inside a project downloads just that project. 📂 Import JSON loads them back (merge or replace).
  • 🤖 Copy for AI copies the whole project as JSON so you can paste it to an AI assistant for help with your content.
