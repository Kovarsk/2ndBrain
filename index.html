📖 User manual

What is this?

Second Brain is a personal workspace that lives entirely in your browser. You organize projects, and inside each project a tree of folders and files. Every file holds a piece of HTML or Markdown that gets rendered for you.

Projects (this screen)

  • + New project creates a project. Click a card to open it.
  • The ⋯ on a project card lets you open, rename, export, merge or delete it.
  • ⇄ Merge projects combines two projects: pick the main project and the one to merge into it. Contents are smart-merged (matched by name, conflicts resolved by you), and you can optionally delete the source project afterwards.

The tree (left sidebar)

  • Click a file → its HTML is rendered full-size on the right.
  • Click a folder → the right side renders everything inside it: each file becomes a block, and each subfolder becomes its own block containing its files (and its subfolders, recursively).
  • In the rendered view, the ▾ caret on a block collapses/expands it, and clicking the name of a file or folder jumps to it. Folder blocks show the file count plus three status dots with how many files inside are to do / in progress / done.
  • The ▸ / ▾ caret only expands or collapses a folder in the tree — it doesn't change the selection.
  • The ⋯ menu on any item: new file / new subfolder (folders), edit HTML and status (files), rename, move, reorder, delete.
  • The dot next to a task is its status — click it to cycle: to do → in progress → done. Information files (📘) have no dot.

Creating files & folders — where do they go?

  • The + Folder and + File buttons at the bottom of the sidebar create inside the currently selected folder.
  • If nothing is selected (or a file is selected), they are created at the project root.
  • To create inside a specific folder regardless of selection, use that folder's ⋯ menu → + New file / + New subfolder.
  • Placed something in the wrong spot? Use ⋯ → Move… to relocate it anywhere in the tree.

Editing file content

  • Open a file and press ✏️ Edit (or ⋯ → Edit content). Each file is either HTML or Markdown — switch with the toggle at the top of the editor. 👁 Preview shows the result before saving.
  • The toolbar above the editor inserts common elements for the current format: headings, bold/italic, lists, checkboxes, links, images, tables, quotes, code and more. Select text first to wrap it, or click with nothing selected to insert a template. In HTML mode, 📄 Page inserts a full page skeleton and Size… wraps the selection in a chosen font size.
  • Markdown supports headings, bold/italic, links, images, code blocks, nested lists, task checkboxes - [ ], tables, quotes and dividers. Name a new file ending in .md to start it in Markdown mode.
  • Files render in a sandboxed frame, so scripts inside your HTML run safely per file. Drag the bottom edge of a rendered block to resize it.

Tasks, information files & details

  • Every file is either a task or an information file — you choose when creating it, and can convert later via the ⋯ menu. Only tasks have a status; info files (📘) are reference material and are ignored in all progress counts.
  • ⋯ → Details… on a file: add tags to any file; on tasks also set a deadline (shown as a chip, red when overdue), pick dependencies (other tasks it needs) and connect related information files. Click the ⛓ or 📘 chip on a rendered task to jump to its dependencies or linked info.

Internal links

In the editor, select some text and press 🧠 to turn it into a link to any file or folder in the project. Clicking it in the rendered view jumps straight there. Links are standard blue by default — change the color under 🎨 Theme → "Internal links".

Link previews

If a file's content is just a URL (nothing else), the page itself is embedded as a live preview, with an open ↗ link above it. Some sites refuse to be embedded and stay blank — use open ↗ for those.

Mirrors — live views across projects

  • Use the 🪞 button at the bottom of the sidebar (or a folder's ⋯ → Mirror from a project…) to embed a live view of a folder or file from another project.
  • Mirrors always show the current content of the source: edit at the source (the ↗ Source button jumps there) and the mirror updates automatically. Deleting a mirror never touches the source.

Minimizing the menu

The « button next to the project name shrinks the sidebar to icons only, plus a mini-path with the last two folders and the current file — handy on mobile. Press » to expand it again.

Themes

🎨 Theme (inside a project) sets that project's colors — pick a preset or tune each color individually, including the three status colors (to do / in progress / done). On the Projects screen, each card's top border shows overall progress: red (nothing done) → yellow (halfway) → green (all done). Projects without any tasks get a blue border, and card counts only look at tasks — information files are ignored.

Saving, backup & AI

  • Everything saves automatically to this browser's local storage. It never leaves your machine — but clearing browser data (or switching browser/device) loses it, so export regularly.
  • 💾 Export all downloads the whole workspace as JSON; 💾 Export inside a project downloads just that project. 📂 Import JSON loads them back. When importing a workspace on the Projects screen you can Add all (simple append, may create duplicate projects), Smart merge (projects are matched by name; inside matching projects, folders and files are matched by name too — new items are added and for files that exist on both sides you choose which version to keep), or Replace the whole workspace. When importing into a project the same choices apply to that project's tree.
  • 🤖 Copy for AI copies the whole project as JSON so you can paste it to an AI assistant for help with your content.
